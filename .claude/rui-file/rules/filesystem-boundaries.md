# rui-file 文件组织契约

> 提供目录结构整理与重复文件检测能力，所有变更与确认边界在此固化以保护用户数据。

## 角色定位

| 维度 | 取值 |
|------|------|
| 输入 | 用户选定的目标目录（默认 `~/Downloads`） |
| 输出 | 经用户确认后的文件移动 / 重命名 / 删除提案 |
| 调用模式 | 对话式，无 CLI 入口 |
| 状态 | 仅在用户明确 yes/no/modify 之后执行，否则为只读审计 |

## 输入契约

| 字段 | 来源 | 必填 | 缺省 |
|------|------|:---:|------|
| 目标目录 | 用户指定 | 是 | 询问用户 |
| 整理目标 | 用户口述（"找重复"、"按类型分"、"按时间"）| 是 | 询问用户 |
| 排除项 | 用户列出 | 否 | 询问"哪些不能动" |
| 整理力度 | 用户选择（保守/标准/激进）| 否 | 默认"保守" |

## 输出契约

| 产物 | 模式 | 风险级别 | 确认要求 |
|------|:---:|:---:|---------|
| 当前结构分析报告 | 文本输出 | 低 | 无需确认 |
| 重复文件 hash 清单 | 文本输出 | 低 | 无需确认 |
| 组织计划提案 | markdown 表格 | 中 | 必须 yes/no/modify |
| 文件移动 / 重命名 | mutate | 中 | 必须 yes |
| 文件删除 | mutate | **高** | 必须 **三次确认**（写明文件名）|

## 路径所有权矩阵

| 路径 | 权限 | 理由 |
|------|:---:|------|
| 用户指定的目标目录 | propose-then-write（确认后） | 用户授权范围内 |
| `~/` 之外的目录 | **never write** | 防止越权 |
| 系统目录（`/etc`、`/usr`、`/Library`、`/System`）| **never read** | 安全 |
| 当前工作项目 | 默认排除（在用户未明确允许前） | 防止误改活跃项目 |
| 任何重命名 / 删除路径 | 必须用户明确 yes | 不可逆 |

## 操作分类与风险

| 操作 | 风险 | 回滚 | 确认要求 |
|------|:---:|------|:---:|
| 列出文件 / 统计 / 类型分布 | 低 | N/A | 无 |
| 重复检测（hash / 文件名 / 大小）| 低 | N/A | 无 |
| 文件移动（同一文件系统）| 中 | 可移动回去 | 一次 yes |
| 文件重命名 | 中 | git 可追踪；裸目录需重命名脚本回滚 | 一次 yes |
| 文件删除 | **极高** | 不可逆 | **三次确认 + 文件名核对** |
| 跨设备移动（外接磁盘到本地） | 中 | 用户手动 | 一次 yes |
| 归档压缩 | 中 | 解压 | 一次 yes |

## 自我隔离硬规则

| # | 规则 | 设计理由 | 违反处置 |
|---|------|---------|---------|
| 1 | 每个不可逆操作（删除 / 重命名）必须列出文件名让用户确认 | 防止误操作 | 阻塞操作 |
| 2 | 不操作用户明确排除的路径（在对话前就已声明）| 尊重用户意图 | 永久跳过 |
| 3 | 不解读文件内容做语义判断 — 只按扩展名 / 名称 / 日期 / hash 分组 | 不读取用户敏感数据 | 报告越界 |
| 4 | 不调用云同步 / 网盘 / 备份命令 | 本地边界 | 拒绝 |
| 5 | 不修改外接磁盘、SMB 挂载、`.git/`、`node_modules/` 内的真实内容 | 防跨设备破坏 | 跳过或仅只读 |
| 6 | 不读取 / 移动「点文件」（`.git`, `.ssh`, `.env`, `.config` 等）| 保护配置 | 列名跳过 |
| 7 | 不批量删除超过 10 个文件而不逐一询问 | 防误删大目录 | 强制逐项确认 |

## 失败与降级

| 场景 | 行为 | 恢复方式 |
|------|------|---------|
| 目录无读权限 | 报告并请求 `ls -la` 手动确认 | 用户 sudo / chmod |
| 移动目标路径已存在同名文件 | 跳到下个编号 `_1`、`_2` ... | 提示用户决策 |
| 磁盘剩余空间不足 | 停止提议移动方案 | 用户清理空间 |
| 文件正在被占用 | 跳过并报告（Windows / macOS 上偶发）| 用户关闭占用进程 |
| 软链接 / 硬链接 | 不跟踪链接目标，避免环 | 仅对实际路径操作 |
| 重复文件 hash 冲突但 mtime 不同 | 报告而非自动决定 | 让用户挑哪个保留 |

## 与兄弟技能的关系

| 技能 | 方向 | 接口 | 说明 |
|------|------|------|------|
| (无) | — | rui-file 独立调用 | 不消费 / 不被消费 rui-* 产物 |

## 集成点

> 当前状态：本技能无自动消费者，无管线级集成。每次调用由用户在对话中显式触发（"Help me organize Downloads"、"Find duplicates in Documents"）。如有"下载完成后自动整理"的需求，应在用户的下载器侧（如 [[rui-video]] 选配）显式调用，而非让 rui-file 反向监听。
