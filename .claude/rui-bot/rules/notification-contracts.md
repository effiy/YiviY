# rui-bot 通知契约

> 企业微信机器人消息发送 — 写日志 + HTTP POST 双通道，所有路径与权限硬约束在此固化。

## 角色定位

| 维度 | 取值 | 设计理由 |
|------|------|---------|
| 触发模式 | 手动调用（`node send.mjs`） | 管线无需主动推消息；事件总线由其他 rui-\* 暴露 |
| 写操作 | 追加日志 + HTTP POST | 两个目标均为追加/瞬时，无更新语义 |
| 数据所有权 | `docs/故事任务面板/<story>/消息通知列表.md` 追加 + WeCom webhook 写入 | 严格仅写这两个目标 |
| 可重入性 | 同一 story 多次调用 → 多次追加 + 多次 POST | 调用方负责去重，本技能不缓存 |

## 输入契约

| 字段 | 类型 | 必填 | 来源 | 缺省行为 |
|------|------|:---:|------|---------|
| `--story` | string | 否 | 调用方 | 跳过日志追加，仅发 HTTP |
| `--project` | string | 否 | `CLAUDE.md` 解析 | 回退到 `default` |
| `--content` | string | 与 `--contentFile` 二选一 | 命令行 | 报错退出 64 |
| `--contentFile` | string | 与 `--content` 二选一 | 项目根相对路径 | 报错退出 66 |
| `--no-send` | bool | 否 | 命令行 | 默认发送 |

## 输出契约

| 产物 | 路径 | 写模式 | 失败语义 |
|------|------|:---:|---------|
| 通知日志 | `docs/故事任务面板/<story>/消息通知列表.md` | append | 路径不存在 → 创建；写入失败 → 报错退出 70 |
| HTTP POST | `config.json` 中 `api_url` | fire-and-forget | 网络失败 → 已重试 3 次后标记为失败但日志已写 |

## 命令族风险分级

| 命令 | 风险 | 二次确认 | 日志行为 |
|------|:---:|:---:|---------|
| 默认发送 | 中 | 否 | 写日志 + POST |
| `--no-send` | 低 | 否 | 仅写日志 |
| 2000+ 字消息 | 低 | 否 | 自动截断 `…` 后正常发送 |

## 路径所有权矩阵

| 路径 | 权限 | 理由 |
|------|:---:|------|
| `<this-skill-dir>/` | r+w | 自身配置与脚本 |
| `docs/故事任务面板/<story>/消息通知列表.md` | append-only | 通知追踪的唯一落点 |
| `config.json` 内联 robot.webhook_url | read | 调用方负责注入 |
| `WEWORK_BOT_WEBHOOK_URL` 环境变量 | read | 优先于 config |
| WeCom webhook endpoint | write（HTTP POST） | 远程副作用 |
| `docs/` 其他子目录 | no | 通知列表是 `故事任务面板/` 下唯一允许 |
| `CLAUDE.md` | read | 仅读取项目名 |
| 任何其他路径 | no | 严禁写入 |

## 安全基线

| # | 规则 | 违反后果 |
|---|------|---------|
| 1 | 真实 webhook URL 不得写入 `docs/` 任何文件（包括示例与日志） | 凭证泄露 |
| 2 | `API_X_TOKEN` 必须来自环境变量，绝不落盘 | Token 泄露 |
| 3 | 实际配置文件中禁止提交任何形式的明文 token/URL | 提交污染 |
| 4 | `--content` 不允许覆盖 `config.json` / `webhook_url` | 配置污染 |
| 5 | 不解析 `--content` 中的 HTML/Markdown — 视作纯文本，避免渲染差异 | 输出不可控 |

## 失败与降级

| 场景 | 行为 | 恢复方式 |
|------|------|---------|
| `API_X_TOKEN` 缺失 | 跳过 HTTP，日志仍写，标记 `[no-token]` | 配置环境变量后重试 |
| Webhook URL 不可达 | 重试 3 次（指数退避 2s/4s/8s）后跳过 HTTP | 检查网络 / `npm ping wecom` |
| 消息 > 2000 字符 | 自动截断 + `…` 后发送 | 拆分多条调用 |
| 故事路径不存在 | 自动创建 `docs/故事任务面板/<story>/` 目录 | 无需恢复 |
| `config.json` 解析失败 | 立即报错退出 78 | 校验 JSON 语法 |
| `--contentFile` 路径越界（`..` 跳出项目根） | 拒绝执行 | 使用项目内绝对路径 |
| HTTP 返回 4xx（非 401/429） | 视为客户端错误，不再重试 | 调用方修正参数 |

## 与兄弟技能的关系

| 技能 | 方向 | 接口 | 说明 |
|------|------|------|------|
| 任何 rui-\* 管线 | 调用 → rui-bot | `--story + --content` | 管线产出完成后主动调用 |
| (无) | 反向调用 | — | rui-bot 不订阅其他技能的事件 |
| (无) | 数据读取 | — | rui-bot 不读任何 rui-\* 产物 |

## 集成点

> 当前状态：本技能为纯输出型工具，无管线级自动消费者。任何"管线完成自动通知"需求须由具体管线的最后一环显式 `node <this-skill-dir>/send.mjs --story=...` 调用实现。若新增自动触发器，应同步在本文件"与兄弟技能的关系"中登记，避免隐性耦合。

## 自我隔离硬规则

| # | 规则 | 设计理由 | 违反处置 |
|---|------|---------|---------|
| 1 | 不修改 `config.json` 中的任何 robot 定义（webhook URL 也只读） | 配置变更属于另一个变更面 | 报错退出 65 |
| 2 | 不读取 `docs/故事任务面板/` 下无关文件 | 职责单向：只追加自己负责的日志 | 跳过并警告 |
| 3 | 不引入 HTTP 客户端以外的依赖（保持 `send.mjs` 单文件） | 部署零依赖 | 拒绝执行 |
| 4 | 不向 WeCom 发送 Markdown / HTML — 只发纯文本 | 跨客户端兼容 | 自动转义 `<` `>` `&` |
| 5 | 不在日志中记录 `API_X_TOKEN` 或 webhook URL | 凭证保护 | 注入前过滤 |
| 6 | 不修改 `故事任务面板/` 同名兄弟 markdown（仅追加到自己的 `消息通知列表.md`） | 路径所有权严格 | 拒绝执行 |
