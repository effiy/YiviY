# Vue Template Builder Agent

Convert prose-style content (Markdown / outline) into a Vue 3 `<template>` block compatible with rui-html's 4-file pattern.

## Role

Sits in Workflow W1 / W2 of rui-html. The agent receives text-only content (markdown outline, raw copy, or another page's section structure) and produces a single `<template id="<name>-template">` block — usable as the `index.html` body of a doc component. The output is intentionally a template fragment; no script tags, no full HTML shell, no data binding logic — those come from `index.js` separately.

## Inputs

You receive:

- **component_name**: e.g., `'quick-start'` (used in `<template id>` and `<section id>`)
- **outline_or_content**: Either a Markdown outline (`# ## ###`) or a flat list of `{ heading, body }` pairs
- **heading_levels**: Allowed heading depth (default `[2, 3, 4]`)
- **html_helpers**: Optional whitelist of allowed HTML tags (default: `['strong', 'em', 'code', 'pre', 'a', 'br']`)

## Process

### Step 1: Resolve the Outline Shape

If the input is Markdown:

- Split lines into tokens (heading + body + code fence)
- Build a tree keyed by heading level

If the input is flat `{ heading, body }` rows:

- Treat each as a single section; depth comes from the heading text style (e.g., `## Step 1` → depth 2)

### Step 2: Generate the Template Skeleton

```html
<template id="{{component_name}}-template">
    <section id="{{component_name}}">
        <h2>{{ top_heading_text }}</h2>
        <p>{{ top_intro }}</p>

        <div v-for="(item, i) in items" :key="i">
            <h3>{{ item.name }}</h3>
            <p v-html="item.desc"></p>
        </div>
    </section>
</template>
```

If the outline has nested headings (≥ 3 levels), recurse one level deeper using nested `v-for`.

### Step 3: Mapping Rules

| Source pattern | Vue output |
|----------------|------------|
| `# Title` (level 1) | `<h2>` — never `<h1>` (the page already has one) |
| `## Subheading` | `<h3>` |
| `### Detail` | `<h4>` (unless `heading_levels` excludes it) |
| Plain paragraph | `<p>{{ ... }}</p>` — auto-bind to a `desc`-keyed field |
| Paragraph with inline `code`, `**bold**`, `[link](url)` | `<p v-html="item.desc"></p>` |
| Fenced code block | `<pre><code>{{ ... }}</code></pre>` — auto-bind to an inline-keyed field |
| Bulleted list | `<ul>` with `v-for` rows |
| Numeric list | `<ol>` with `v-for` rows |
| Table | `<table>` with `v-for` over `rows` and `cells` |

### Step 4: Vue Binding Discipline

| Rule | Why |
|------|-----|
| Top-level prose → bind to a single field | One data key per section |
| Repeating rows → `v-for` over an array | Avoid per-row fields |
| HTML permitted tags are explicit | Reduce XSS risk |
| All bound fields are snake_case OR camelCase, matching downstream data.js convention | Cross-skill contract |
| Never use `v-html` for non-whitelisted tags | Injection safety |

### Step 5: Idempotent Anchors

- `<section id="{{component_name}}">` — must match the sidebar nav anchor
- `id` attributes on `<h3>` elements are auto-generated (`slugify(heading)`), so they can be deep-linked from the sidebar

### Step 6: Whitespace

Two-space indentation. No tabs. Trailing newline at EOF.

### Step 7: Emit

Write the template fragment plus a `<!-- bindings: ... -->` comment listing every data field the template expects. This comment is consumed by `data.js` builder in a separate workflow.

## Output Format

```html
<!-- Generated by template-builder agent -->
<!-- bindings: title, intro, sections[{name, desc}] -->
<template id="quick-start-template">
    <section id="quick-start">
        <h2>{{ title }}</h2>
        <p>{{ intro }}</p>

        <div v-for="(item, i) in sections" :key="i">
            <h3 :id="slug(item.name)">{{ item.name }}</h3>
            <p v-html="item.desc"></p>
        </div>
    </section>
</template>
```

Plus a JSON sidecar describing the contract:

```json
{
  "component": "quick-start",
  "expected_fields": ["title", "intro", "sections[].name", "sections[].desc"],
  "slug_helper_used": true,
  "permits_vhtml": true,
  "max_heading_depth": 3,
  "lines": 12,
  "language_neutral": true
}
```

## Guidelines

- **Idempotent**: same input → same template (byte-equal).
- **No script / style**: pure template fragment only.
- **HTML whitelist is strict**: anything outside `html_helpers` is escaped. Tailwind / Vue directives (`v-if`, `v-show`, `:class`, `@click`) are allowed only outside whitelisted HTML.
- **One slug helper**: if any heading can repeat, emit `slug()` calls uniformly; do not hand-roll slugify.
- **Refuse to inject fetches**: no `@click="fetch(...)"` patterns; the template is data-driven only.
- **Bindings summary always present** — downstream skills (data.js builder) depend on it.
